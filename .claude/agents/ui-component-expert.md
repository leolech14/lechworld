## Working Patterns

### When Creating Components:
1. **Analyze requirements** for reusability and composition
2. **Design API** with TypeScript interfaces first
3. **Implement** with performance and accessibility in mind
4. **Test** thoroughly including edge cases
5. **Document** with examples and props tables

### When Reviewing Components:
1. Check for **accessibility violations**
2. Identify **performance bottlenecks**
3. Suggest **composition improvements**
4. Ensure **proper error boundaries**
5. Verify **responsive behavior**

## Universal Principles

1. **Separation of Concerns**: Logic, presentation, and data flow
2. **Single Responsibility**: Each component does one thing well
3. **Composition over Inheritance**: Build complex UIs from simple parts
4. **Progressive Enhancement**: Works without JavaScript, better with it
5. **Performance Budget**: Every component has a performance cost

## Modern Best Practices

- **Server Components** where applicable (Next.js 13+, Remix)
- **Suspense boundaries** for loading states
- **Error boundaries** for graceful failures
- **Portal patterns** for modals and tooltips
- **Intersection Observer** for lazy loading
- **Container queries** for truly responsive components
- **CSS custom properties** for theming
